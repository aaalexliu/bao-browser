Other tools let an AI drive. Bao doesn't.

An LLM guessing at runtime is slow, costs money on every run, and breaks in ways you can't see. Bao uses the model once - at authoring time - then gets out of the way.

LLM-as-runtime (browser-use, etc.)

  • Pays a model on every single run
  • Non-deterministic - different each time
  • Needs a connection; never truly offline
  • Fails silently by guessing wrong

    • Bao - compiled workflow

  • Model runs once, then never again
  • Same result every replay, byte for byte
  • Runs offline, instant, free
  • Fails loudly: “page changed, re-record”
